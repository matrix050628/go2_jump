# 研究路线

[English](research_program.md)

## 1. 当前研究假设

这个项目当前最值得投入的方向，已经不是纯模板控制，也不是纯端到端策略。

当前工作假设是：

高层输出显式跳跃意图，低层用接触感知的 MuJoCo-native whole-body MPC 执行，这条混合路线有机会同时获得更好的可解释性、鲁棒性和迁移性。

长期目标结构是：

`RL planner -> explicit JumpIntent -> contact-aware MuJoCo-native MPC -> LowCmd`

## 2. 阶段 A：先把接口打牢

目标：

在追性能之前，先把控制栈做成“可观测、可替换、可对比”的系统。

阶段交付物：

- 稳定的 `JumpTask` 接口
- 稳定的 `JumpIntent` 接口
- 能记录 active intent 和 backend 的控制器诊断链路
- 可复现实验的单次试验和批量试验脚本
- 可以一键切换 `no_intent` 和 `explicit_intent` 的 A/B 入口

这一阶段的意义是防止后续 RL 和 MPC 迭代变成不可控的盲调。

## 3. 阶段 B：先做强一点的显式 planner 基线

目标：

在引入 RL 之前，先把显式 planner 的接口和基线行为做扎实。

近期工作包括：

- 整理当前启发式 planner
- 保证 task 重建和 intent 重建的一致性
- 先区分“planner 问题”和“low-level 执行波动”再去调参数
- 在理解 low-level 波动来源后，再决定 planner 应该怎么改

这个阶段不是为了永远保留启发式 planner，而是为了建立稳定基线和稳定接口边界。

## 4. 阶段 C：接入 RL 高层 planner

目标：

用学习策略替换启发式 planner，但仍然输出可解释的 kinodynamic jump intent。

RL 预期输出包括：

- 起跳速度目标
- 相位时长目标
- 机身姿态目标
- 收腿与落地准备参数
- 可选的足端偏置参数

RL 输出仍然要受约束：

- 关节范围可行性
- 起跳与落地时序可行性
- 摩擦约束下的意图边界
- 机身俯仰和触地稳定目标

也就是说，RL 负责提升“意图选择”，而不是跳过显式控制结构直接输出底层命令。

## 5. 阶段 D：继续增强 low-level MPC

目标：

把当前 MuJoCo rollout controller 逐步增强成更强的 low-level 执行器。

近期重点包括：

- 提高预测接触与真实接触的一致性
- 提高起跳与触地时机处理的质量
- 加强前后腿载荷重分配
- 改进短时域 cost，使其更关注腾空前移和落地质量

中期可能的增强方向：

- 显式引入 centroidal momentum 项
- 更强的接触力优化
- 更紧的 whole-body 力到关节一致性约束

## 6. 阶段 E：固定评测协议

在进入大规模调参与训练前，需要先固定评测协议。

核心指标：

- 成功率
- 总前移距离
- 腾空阶段前移距离
- 落地后补位距离
- 触地瞬间俯仰角和俯仰角速度
- 峰值力矩与力矩利用率
- 相位干净程度和接触一致性

核心对比：

- 不使用高层 intent
- 启发式显式 planner
- RL 显式 planner
- 可现实复现的成熟公开方法

## 7. 论文方向

如果这条路线最后做得足够好，论文故事不应该只是“Go2 会跳”。

更有力度的故事应当是：

- Go2 上按目标距离条件化的前跳
- 大部分前移由腾空阶段完成
- 高层意图显式、可解释
- 低层执行由接触感知的 whole-body MPC 完成
- 这是一条同时区别于手工模板控制和黑箱端到端策略的混合路线
